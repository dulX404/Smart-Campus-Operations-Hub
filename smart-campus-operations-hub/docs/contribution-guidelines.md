# Contribution Guidelines

## Team Rules

- Commit frequently in small logical steps
- Use clear and meaningful commit messages
- Open pull requests for review before merging
- Keep changes focused on your assigned module
- Avoid pushing unfinished breaking changes to `dev`

## Suggested Commit Message Style

- `feat: add booking module starter`
- `fix: correct resource route`
- `docs: update setup guide`
- `chore: add ci workflow`
